这篇不再只是介绍 `Lerobot` 项目本身，而是整理我这边相对社区主线做过的主要贡献。

[Lerobot](https://github.com/huggingface/lerobot) 原本更像一个面向机器人数据集、训练与推理的通用框架；我这边的工作重点，则更偏向把它往 **FR3 实机、数采闭环、训练推理、相机接入和仿真验证** 这些真实工程问题上推进。

一句话概括：

> 这部分贡献的核心，不是多支持几个模型名字，而是把一条面向 FR3 的工程闭环真正补出来。

## 主要贡献

### 1. FR3 实机闭环

这一块是最核心的增量。

- 增加了 `Franka Research 3` 的最小接入能力
- 补齐了 `teleop`、`hardware smoke`、`move_to_start`、`runtime` 这类实机调试工具
- 对 OTG、命令滞后、前一帧命令契约等问题做了专门诊断
- 把 FR3、本体夹爪和外接夹爪之间的联动链路往可验证方向收紧

这部分的价值不是“又多接了一种 robot”，而是让 `Lerobot` 在 FR3 上开始具备真实实验所需的工程骨架。

### 2. 数采、回放和坐标对齐

相对社区主线，这边在数据链路上补了很多实战里才会反复踩坑的环节：

- `ee2ee` 动作记录和配套修正
- `FR3 DAS replay` 的真实回放与运行时回放工具
- episode 起始稳定、reset 节奏、SpaceMouse idle gating 等录制细节
- dataset viz 远程查看、时间戳锚定和 episode 切换能力
- `SLAM frame contract`、四元数约定、变换链修正等坐标对齐问题

这部分的结果是：采集、回放、排查和复现实验不再高度依赖手工经验，而是逐步收敛成可重复的工具链。

### 3. 触觉接入和 ACT 训练链路

除了视觉链路，我这边还把 `触觉 + policy` 往前推了一步：

- 增加了 `Paxini Gen2 Omega` 触觉接入
- 给 FR3 的 ACT 增加了触觉编码相关支持
- 补了 `mask2ee`、`partial xyz masking`、`qoff`、`quantile normalization` 等实验配置
- 增加了训练、推理和离线分析配套工具，而不是只留一份 config

也就是说，这里不是只在“模型列表”里再加一项，而是试图把 `policy config -> dataset -> infer runtime -> replay analysis` 这条链做实。

### 4. Hikrobot 相机和多模态采集

另一条很明确的贡献线，是把 `Hikrobot` 相机支持从零散脚本推进成可用模块：

- 集成了相机驱动与配置
- 增加了 `GigE` 发现、录制、静态配置导出工具
- 修了设备采集时间戳可视化问题
- 增加了 handheld multimodal capture 支持

这说明仓库的目标已经不只是“跑标准 dataset”，而是开始承接更接近现场形态的观测接入和录制工作。

### 5. Docker 和部署环境

这边在 `docker/` 和工具链上也补了不少工程工作：

- 增加了用户态和内部态的 Dockerfile
- 增加了训练和主机部署相关的 compose 配置
- 把 FR3 runtime、native toolchain、host env setup 收到同一套工作流里
- 连同 Hikrobot 依赖和实机工具一起纳入部署环境

这部分价值在于：训练、推理、采集和相机依赖不再需要每台机器重新手配一遍。

### 6. MuJoCo 仿真和可观测调试

仿真侧的工作也不只是“能 replay 一次”：

- 增加了 `fr3_mujoco` 环境和 teleop 支持
- 增加了 sim bridge、viewer teleop 和 replay 校验工具
- 增加了多视角相机渲染
- 增加了基于 HTTP MJPEG stream 的仿真画面输出
- 补了 env 和 script 级别的测试

这让仿真不再只是离线看轨迹，而更接近实机前的验证环境。

## 整体取向

如果只看社区主线，Lerobot 更像一个通用开源框架。

如果看我这边的贡献，重点会更偏向：

1. 把实机控制、数采、回放、训练、推理和仿真连成闭环。
2. 把触觉、工业相机和 runtime 工具带入同一套工作流。
3. 让问题可以被复现、诊断和逐步自动化，而不是靠一次次手工调试。

## 我现在对这部分工作的判断

这部分最有价值的地方，不在于“比主线多了多少功能点”，而在于它开始沉淀出一条真实机器人项目会反复用到的底层工程链路：

- 先把设备接进来
- 再把数据录下来
- 再把 replay 和坐标关系对齐
- 再把训练与推理接上
- 最后用仿真和实机一起做验证

这也是我现在继续维护这部分 `Lerobot` 工作的主要意义。
